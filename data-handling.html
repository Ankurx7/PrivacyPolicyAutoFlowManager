Data Storage Implementation

ApplyFlow Manager uses Chrome's Storage API to store user data locally. Specifically, we use:

  • chrome.storage.sync: For storing user profiles and settings that can sync across browsers signed in with the same account

The following storage keys are used:

const STORAGE_KEY_PROFILES = 'smart_job_autofill_profiles';
const STORAGE_KEY_SELECTED_PROFILE_ID = 'selectedProfileId';
const STORAGE_KEY_JOB_SEARCH_HISTORY = 'jobSearchHistory';
const STORAGE_KEY_LAST_JOB_SEARCH = 'lastJobSearch';

Data Transfer & Permissions Explanation

No Data Transfer: ApplyFlow Manager does NOT transmit any collected data outside your browser. All data operations occur locally on your device.

Here is an explanation of each permission requested and why it's needed:

Permissions

Permission Purpose Data Access
storage Store profile data and settings Only accesses its own storage area
activeTab Access form content on pages you're visiting Only when you activate the extension
scripting Inject form-filling scripts Only when auto-filling is activated
tabs Open job search results, detect forms Basic tab information only
contextMenus Add right-click menu options No data access, creates UI elements
notifications Display status notifications No data access, only creates notifications
host_permissions Access job site content Only while actively browsing job sites

Code Examples: How Data Stays Local

Our extension is designed to keep all data local. Here are some code examples that demonstrate this approach:

Saving Profiles Locally

async function saveProfiles(profiles) {
  try {
    await chrome.storage.sync.set({ [STORAGE_KEY_PROFILES]: profiles });
    return true;
  } catch (error) {
    console.error('Error saving profiles:', error);
    return false;
  }
}

Getting Profiles Locally

async function getProfiles() {
  try {
    const result = await chrome.storage.sync.get(STORAGE_KEY_PROFILES);
    return result[STORAGE_KEY_PROFILES] || [];
  } catch (error) {
    console.error('Error getting profiles:', error);
    return [];
  }
}

As you can see from these examples, all data operations use chrome.storage.sync, which stores data locally in the browser and optionally syncs across the user's devices (when signed into Chrome/Edge). There are no API calls to external services.

Data Deletion

Users can delete their data in the following ways:

  • Using the profile management interface in the extension to delete individual profiles
  • Clearing browser extension data through browser settings
  • Uninstalling the extension (which removes all associated storage)
